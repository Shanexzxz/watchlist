## 1. 背景与目标

内容金融组（投资并购部）需要每个交易日向相关同事展示 **二级市场持仓 Watch List**：以表格形式展示组内关注标的当日行情、估值，以及内部持仓信息（MA 持有价值、MoC、模拟仓、TPP 持仓标记）。

### 优先级

- **Phase 1（当前）**：PC 端 Web 页面 — 用户通过浏览器访问，支持多组切换、搜索、排序。
- **Phase 2（后续）**：邮件自动发送 — 作为 Web 页面的补充通道，定时推送 T-1 数据快照。

### 目标

1. 数据自动化：行情类数据来自 **Bloomberg**，持仓类数据来自 **TIM**，无需人工填值。
2. 渲染标准化：前端按统一数据契约渲染，视觉对齐 demo。
3. 权限隔离：8 个行业组各自独立，用户默认只看本组；部分用户可跨组查看。
4. Web 交互：支持搜索/筛选和列排序，提升数据查阅效率。
5. （Phase 2）每交易日上午 9 点定时发送 T-1 数据邮件，收件人可配置（一个行业组一封）。

### 非目标（本期不做）

- 不做交互式 Web 应用（这是**只读数据展示页面**）。
- 不做行情详情弹窗、K 线图等深度交互。
- 不做导出功能（本期）。

## 2. PC 端 Web 页面

### 2.1 访问权限模型

- 用户来自 **8 个行业组**，每个组有各自的 Watch List（标的清单、行业分类独立维护）。
- 8 个组分别为：**内容金融组、消费零售组、智能应用投资组、投资组合管理组、海外投资组、医疗健康组、全球娱乐组、技术企业组**。
- **默认权限**：用户只能查看自己所属组的 Watch List。
- **跨组权限**：部分用户（如管理层、跨组研究员）可查看多个组。
- **UI 表现**：用户可访问的组以 **Tab** 形式展示在页面顶部；无权限的组不显示 Tab。
- 用户信息区显示当前用户名及可访问组数（如"可访问 3/8 组"）。

### 2.2 页面结构

自上而下：

```
┌──────────────────────────────────────────────────────────────┐
│  Watch List                              [头像] 用户名 ▾      │ ← 顶栏（logo + 用户信息）
├──────────────────────────────────────────────────────────────┤
│  [内容金融组] [消费零售组] [智能应用投资组] [投资组合管理组] [海外投资组] [医疗健康组] [全球娱乐组] [技术企业组] │ ← 组切换 Tab
├──────────────────────────────────────────────────────────────┤
│  搜索...  行业: 全部▾                  Jun 17, 2026 (T-1)   │ ← 工具栏
├──────────────────────────────────────────────────────────────┤
│  【内容金融组】Watch list on Jun 17, 2026                      │ ← 标题（含日期）
│  • Market cap over USD10 billion (rose/declined >5%): ...    │ ← 摘要 bullets
│  • Market cap under USD10 billion (rose/declined >10%): ...  │
│  • Upcoming Earnings (HKT): ...                              │
├──────────────────────────────────────────────────────────────┤
│  指数表（9 列，可排序）                                        │
├──────────────────────────────────────────────────────────────┤
│  【电子银行和钱包】（12 列，可排序）                             │
│  【信贷】（12 列，可排序）                                      │
│  ...更多行业表                                                │
├──────────────────────────────────────────────────────────────┤
│  Source: Bloomberg, TIM                                       │ ← 数据来源
│  1. ... 2. ... 3. ...（脚注）                                   │
└──────────────────────────────────────────────────────────────┘
```

- 切换 Tab 后，页面内容整体更新为该组的 Watch List（标题、摘要、表格、脚注均变化）。
- Tab 数量超过屏幕宽度时，支持横向滚动。

### 2.3 交互功能

| 功能 | 说明 |
|---|---|
| **组切换 Tab** | 点击 Tab 切换到对应组的 Watch List；仅显示有权限的组 |
| **搜索** | 输入框支持按公司名或 BBG 代码搜索，实时过滤当前组所有表格行 |
| **行业筛选** | 下拉选择行业分类，仅显示该行业的表格（默认"全部"） |
| **列排序** | 点击表头切换排序方向（升序 → 降序 → 取消），所有数值列均可排序 |
| **日期** | 显示当前数据日期（T-1） |

### 2.4 视觉规范

- 与邮件 demo 保持一致的视觉风格：蓝白交替行（`#fff` / `#C6D9F1`）、黑色 1px 边框、涨绿（`#006400`）跌红（`#c00`）。
- 顶栏：白色背景，"Watch List" logo 蓝色（`#185FA5`），右侧用户头像 + 名称 + 权限信息。
- Tab 栏：白色背景，活跃 Tab 蓝色文字 + 蓝色 2px 底边框，非活跃灰色文字。
- 工具栏：搜索框、筛选下拉、日期标签，白色背景，浅灰边框。
- 表头：浅灰背景（`#e8e8e8`），可点击排序，排序状态用 ↑↓ 图标指示。
- 脚注：与邮件版一致（见 §5）。

## 4. 字段字典

### 4.1 行业表（每只标的 12 列）

> **注意**：当前 8 个组的列结构恰好相同，但后续各组可能需要不同的行情指标。因此列配置是**各组独立**的——某个组增减列不影响其他组。

| # | 列名 | 含义 | 数据源 | 示例 / 格式 |
|---|---|---|---|---|
| 1 | 公司 | 公司名（TPP 持仓加 `*`） | TIM | `Netflix *` |
| 2 | {日期} | 当日收盘价（带币种） | Bloomberg | `USD 97.54` / `HKD 23.46` |
| 3 | Change | 当日涨跌幅 | Bloomberg | `+2.1%`（绿）/ `-1.1%`（红） |
| 4 | Turnover (USD'000) | 当日成交额（千分位，单位千美元） | Bloomberg | `1,846,423` |
| 5 | Adj. P/E (2026E) | 2026 预测市盈率 | Bloomberg | `28.5x` |
| 6 | Mkt Cap (USD M) | 市值（百万美元） | Bloomberg | `45,200` |
| 7 | Price Change (YTD) | 年初至今涨跌幅 | Bloomberg | `+12%` |
| 8 | Price Change (L30D) | 近 30 日涨跌幅 | Bloomberg | `+3%` |
| 9 | Price Change (L365D) | 近 365 日涨跌幅 | Bloomberg | `+48%` |
| 10 | MA 持有价值 (USD M) | 投后持有价值（百万美元） | TIM | `1,500.50` |
| 11 | MoC | Multiple of Capital（含 CB 不含减值） | TIM | `1.25x` |
| 12 | 模拟仓 | 该 ticker 是否在 TIM 模拟仓中 | TIM | `Y` / `N` |

> **MA 持有价值、MoC、模拟仓、TPP 持仓标记** 均来自 TIM；**其余所有市场/行情/估值数据** 均来自 Bloomberg。

### 4.2 指数表（9 列）

仅含上表 **第 1–9 列**（无 MA 持有价值 / MoC / 模拟仓）。首列为指数名，价格/Turnover 等指数无对应值的填 `-`。

> **注意**：指数表的列配置同样各组独立，不同组可能订阅不同的指数和行情字段。

## 5. 底部备注 / 脚注样式

底部区域自上而下两部分：

**① Source 行**
```
Source: Bloomberg, TIM
```
- 紧跟最后一张表格下方，黑字、左对齐、加粗 `Source:` 前缀。

**② 编号脚注列表（Footnotes）**
- 黑字白底、**左对齐、垂直排列**，与表格之间留明显间距。
- 每条以 **数字序号 + `.`** 开头（`1.` `2.` `3.` …），逐条说明。
- 字号 11px，行高放宽（约 1.6–1.7）以便阅读。
- 顶部用 `#ccc` 1px 细线与表格分隔。

脚注示例条目：

```
Source: Bloomberg, TIM

1. Above data were sourced as at 7:00 am on Jun 12, 2026 (HKT).
2. Adjusted P/E ratios are based on EPS ex extraordinary items for calendar year 2026E.
3. Dual-listed company: market capitalisation of secondary listing is calculated with
   shares outstanding implied by the primary listing.
4. For companies newly listed in 2026, L365D / YTD price changes were marked as n.a.
   due to data unavailability.
5. * marks companies in which TPP holds a position (per TIM).
```

## 6. 邮件发送（Phase 2）

- 每交易日上午 9:00 定时发送 T-1 数据。
- 收件人可配置（一个行业组一封）。
- 邮件内容为该组 Watch List 的 HTML 快照（与 Web 页面视觉一致，但不含交互）。
- 邮件数据源同 Web：Bloomberg（行情）+ TIM（持仓）。
